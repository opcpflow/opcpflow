# OpcpFlow Governance

This document outlines the governance model for the OpcpFlow open source project.

## Maintainers

Maintainers are responsible for the overall health of the project, including:

- Reviewing and merging pull requests
- Setting the project roadmap and priorities
- Managing releases
- Enforcing the code of conduct

## Contributors

Anyone can contribute to OpcpFlow! See [CONTRIBUTING.md](.github/CONTRIBUTING.md) for how to get started.

## Decision Making

- **Technical decisions** are made by maintainers through lazy consensus — if no objections are raised within a reasonable period (typically 72 hours for significant changes), a proposal is considered accepted.
- **Roadmap decisions** are made by maintainers, informed by community feedback via GitHub Discussions and issues.
- **Breaking changes** require discussion and a consensus among maintainers before implementation.

## Roles

### Maintainer
- Has commit access to the repository
- Can approve and merge pull requests
- Can create releases
- Expected to participate in project discussions and reviews

### Contributor
- Can submit pull requests
- Can participate in discussions
- Can be promoted to maintainer after demonstrating consistent, high-quality contributions

## Adding Maintainers

New maintainers are added by consensus of existing maintainers. Candidates are typically contributors who have:

- Made significant, consistent contributions over time
- Demonstrated good judgment in code reviews and discussions
- Shown understanding of the project's architecture and goals

## Code of Conduct

All participants in the OpcpFlow community are expected to follow our [Code of Conduct](.github/CODE_OF_CONDUCT.md).

## Questions?

Open a [GitHub Discussion](https://github.com/opcpflow/opcpflow/discussions) for governance questions.
